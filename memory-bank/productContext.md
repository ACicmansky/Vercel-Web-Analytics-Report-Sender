# Product Context

## Problem Statement
Website owners need regular insights into their web analytics but often lack the time to:
- Manually check analytics dashboards
- Interpret raw data into actionable insights
- Track trends over time
- Share reports with stakeholders

## Solution
An automated reporting system that:
- Eliminates manual data collection
- Transforms raw analytics into digestible summaries
- Delivers insights directly to stakeholders' inboxes
- Provides consistent, scheduled reporting

## Target Users
- **Primary**: Website owners/administrators of www.lemolegal.sk
- **Secondary**: Marketing teams, stakeholders requiring analytics insights

## User Experience Goals

### Automation First
- Zero manual intervention required after initial setup
- "Set it and forget it" reliability
- Automatic error recovery where possible

### Clarity & Actionability
- AI summaries should be:
  - Easy to understand for non-technical users
  - Highlight key trends and changes
  - Provide context (comparisons to previous periods)
  - Actionable insights when possible

### Professional Delivery
- Email reports should be:
  - Well-formatted and visually clean
  - Include relevant metrics and charts (if applicable)
  - Professional tone
  - Consistent branding

## Key Metrics to Track
- Page views (total and unique)
- Visitor count (unique visitors)
- Top pages/routes
- Traffic sources (referrers)
- Geographic distribution
- Device/browser breakdown
- Trends compared to previous period

## Value Proposition
1. **Time Savings**: Eliminates manual reporting work
2. **Consistency**: Regular, predictable reporting schedule
3. **Intelligence**: AI-powered insights beyond raw numbers
4. **Accessibility**: Delivered directly to email inbox
5. **Scalability**: Foundation for multi-site reporting in future

## Use Cases

### Primary Use Case: Monthly Analytics Report
1. System triggers on 30-day schedule
2. Fetches last 30 days of analytics data
3. AI analyzes data and generates summary
4. Email sent to configured recipients
5. Log stored for audit trail

### Secondary Use Case: Error Handling
1. API failure or rate limiting occurs
2. System logs error with details
3. Retry logic attempts recovery
4. Notification sent if critical failure

### Future Use Case: Multi-Site Management
1. Support multiple websites
2. Consolidated or separate reports
3. Per-site configuration
4. Comparative analytics across sites
